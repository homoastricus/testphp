Практическое задание It Pelag интернатура (Блог)

Подготовить веб-приложение типа Блог на Php с использованием базы данных MySQL, приложение собой будет представлять блог со следующими функциями

Функционал:
1. Регистрация (добавление) пользователя в БД (страница Регистрация)
2. Возможность добавления статьи через веб-форму (страница Добавить Статью)
3. Возможность редактирования статьи через веб-форму (страница Изменить Статью)
4. Возможность добавления комментария к статье (страница Добавить Комментарий)
5. просмотр статьи и связанных с ней комментариев (страница Просмотр статьи с формой добавления комментария к ней)
6. Пагинация (постраничный вывод статей на странице список статей, она же Главная страница)
Страницы:
1. Регистрация - содержит форму регистрации (имя, пароль)
2. "Добавление статьи" - содержит форму добавления статьи (название, текст). Название должно быть уникально. При попытке добавления дубля статьи по названию - выдавать предупреждение с возвратом на страницу добавления. После успешного добавления статьи происходит автоматический редирект на новую статью (просмотр статьи).
3. Страница "Редактирования статьи" - аналогично странице добавления статьи.
4. "Просмотр статьи" - содержит саму статью (поля название и текст) и ниже расположенные комментарии к ней от пользователей, упорядоченные по дате добавления (старые сверху), с указанием даты добавления комментария. Также бонусом можно добавить функционалов Лайки для статьи.
5. Страница Список Статей (она же Главная страница), содержит постраничный вывод статей название и первые 255 символов текста по 10 шт на странице. Вывод формы авторизации пользователя может быть реализован как отдельный компонент на любой странице, так и на отдельной странице.
Требования к данным (модель):
- Статья: уникальный идентификатор, название статьи (не может быть короче 3х символов и длиннее 255), текст (не может быть пуст), дата и время добавления статьи

- Комментарий: уникальный идентификатор, идентификатор родительской статьи, идентификатор автора статьи, текст комментария (не может быть пуст), дата и время добавления статьи

- Пользователь: уникальный идентификатор, имя (логин, уникальное поле, не может быть короче 3х символов и длиннее 255), пароль пользователя, дата и время регистрации
Требования к функционалу приложению:
- Только автор статьи может редактировать статью.
- Только авторизованный пользователь может добавлять комментарии.
- Авторизация может быть с помощью механизма сессий или через cookie.
- Проверка уникальности названия статьи и логина пользователя
Требования к архитектуре:
- Крайне желательно реализовать приложение через концепцию MVC, не должно быть такого, что логика приложения перемешана с html и с запросами в базу данных (спагетти код)
- желательно написать полностью самописное приложение или же с использованием микрофреймворка (простой роутер, модель, контроллер, вид) Вообще, чем меньше будет написано кода и при этом чем он будет лучше соответствовать концепциям Dry, Kiss, и стандартам Psr, тем лучше.
Критерии приемки результата. При проверке работы будет использоваться балловая система оценки результата. Базовый работающий функционал приложения дает 50 баллов. За дополнительный функционал будут начисляться дополнительные баллы.
1. Реализовать функцию оставить / удалить лайк к статье (лайк должен записываться в БД с привязкой к статье и должна быть возможность оставить и удалить только один лайк для данной статьи текущему пользователю) +10 баллов

2. При удачной попытке написать свой микрофреймворк или же использовании чужого (хорошенько разобравшись в его структуре при этом) +10 баллов

3. При использовании крупных фреймворков типа Laravel дополнительных баллов нет (данное задание не на знание крупных фрейморков, а на проверку умения писать самописный код и создавать архитектуру простых веб-приложений).

4. При соблюдении стандартов psr +7 Баллов

5. При наличии единого общего шаблона (layout) для всех страниц +5 Баллов

6. При наличии возможности запустить сразу на нескольких серверах (встроенный сервер, apache, nginx) +5 Баллов

7. При наличии рабочих юнит-тестов +10 Баллов

8. При наличии грамотной валидации форм +7 Баллов

9. При успешном самостоятельном запуске на тестовом веб-сервере (nginx) +12 баллов

10. Отсутствие базовых уязвимостей в веб-приложении (xss, выполнения javascript в формах, sql-иньекций) +10 баллов

11. Наличие миграции для базового импорта данных в БД +5 баллов

12. Валидная, корректная, "ровная" верстка +7 баллов

13. Грамотно составленный .gitignore и грамотная работа с git +7 баллов

14. Грамотная реализация хранения данных пользователя (исходный пароль не хранится в чистом виде в БД, хранится хеш либо хешированная соль) +5 баллов

15. Защита от флуда (не может быть одним пользователем добавлено чаще одного комментария в минуту к данной статье и больше трех комментариев подряд в одну статью) +10 баллов

Максимум за практическое задание можно набрать 160 баллов. Минимальное кол-во балов для успешной сдачи задания - 50 (для этого требуется полностью работающий минимальный функционал без допфункционала и успешно произвести его презентацию)
Сдача практического задания:
- Предварительный показ 27 апреля (суббота, в любое время с 10.00 и до 17 00, по договоренности), к этому времени должно в общих чертах работать большая часть функционала. Демонстрация с целью получить обратную связь и разобрать трудные вопросы в реализации. - Финальный этап сдачи - 2 мая (четверг), к этому времени ожидается полностью работающий функционал + демонстрация работы приложения. - Первый этап сдачи: устный краткий рассказ с демонстрацией на экране как и что работает, какие компоненты, библиотеки, архитектурные приемы использовались, минут на 5-10
- Второй этап сдачи: Демонстрация работающего функционала приложения, минут 10 по времени
